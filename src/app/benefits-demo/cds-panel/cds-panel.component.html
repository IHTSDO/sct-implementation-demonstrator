This demo uses this open source resource:

SNOMED FHIR Clinical Decision Support Service

https://github.com/IHTSDO/snomed-fhir-cds-service

This demo showcases the SNOMED FHIR CDS Service and is intended for demonstration purposes only, not for real-world clinical use. Built on FHIR standards and SNOMED CT, it analyzes medication orders to detect contraindications, drug interactions, abnormal dosing, and validation errors.

Key Components

  • Rules Engine: Loads medication-condition and drug-drug interaction rules.
  • FHIR Integration: Accepts MedicationRequest, Condition, AllergyIntolerance, and Patient resources.
  • Knowledge bases:

    Interactions: A limited list of medication interaction risks created from publicly available resources for the demo.

    Contraindications: A limited list of medication-condition contraindications created from publicly available resources for the demo.

    DDD: Uses WHO ATC Defined Daily Doses to estimate excessive dosage alerts.

    Thresholds: Supports adjustable dosage alert limits.

Alert Types

  • Contraindication Alerts: Flags when a medication is unsafe for a patient's existing medical conditions. WARNING indicates a critical contraindication, while INFO suggests caution.
  • Medication Allergy Alerts: Flags when a medication conflicts with recorded allergy information.
  • High Dosage Alerts: Identifies when the prescribed daily dose exceeds the WHO Defined Daily Dose reference. INFO indicates 2-4x DDD and WARNING indicates 4x DDD or higher.

    Formula: Prescribed Daily Dose / DDD Reference = Alert Threshold

    Configuration: Acceptable threshold 2x DDD, maximum threshold 4x DDD.

  • Drug-Drug Interaction Alerts: Detects risky medication combinations, with INFO, WARNING, and CRITICAL severities based on interaction strength.
  • Validation Error Alerts: Flags incomplete or invalid prescription data such as invalid dose units, missing frequency, or incomplete information.

Alert Severity Indicators

  • INFO: Informational
  • WARNING: Important safety concern
  • CRITICAL: Severe safety issue
